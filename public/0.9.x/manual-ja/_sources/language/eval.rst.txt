==========
Evaluation
==========

This chapter specifies the operational semantics of :dfn:`abstract instructions`
within an :dfn:`abstract stack machine`,
to indirectly give semantics of a Kink program.
Abstract instructions are transformed from a Kink program
through :term:`desugaring` and :term:`translation` steps.

An :dfn:`abstract stack machine` is an abstract machine
which evaluates :term:`abstract instructions <abstract instruction>`.

An :dfn:`abstract instruction` is an instruction which is evaluated by the :term:`abstract stack machine`.

Multiple abstract stack machines can exist in a single :term:`runtime`,
and they can evaluate :term:`abstract instructions <abstract instruction>` in parallel.

The abstract stack machine and abstract instructions are defined
just to show how Kink programs should behave.
Thus, an implementation of :term:`runtime` need not implement the abstract stack machine literally,
however, the observable behavior must obey the definition.

.. contents::
  :local:
  :backlinks: none

Conventions
===========

Abstract instruction
--------------------

An abstract instruction is parenthesized like the following:

::

  (varref "op_store")

Here, ``varref`` is the opcode of the instruction, and
``"op_store"`` is the operand of the instruction.

Every instruction has a location as an implicit operand.

Data types
----------

The following data types are used as operands of abstract instructions:

* val : a Kink val.
* str : an immutable, finite length sequence of unicode codepoints,
  which can be represented by a :doc:`str value </api/kink-STR>`.
* num : a combination of (Mantissa, Scale),
  which can be represented by a :doc:`num value </api/kink-NUM>`.
* location : a combination of (Program_name, Program_text, Pos),
  which can be represented by a :ref:`location value <api-33da096af45bcf193ac0fbc14b17592f>`.
* a sequence of instructions.

Sequence of instructions
------------------------

A sequence of instructions is represented by just placing instructions in order like the following:

::

  (emptyvec) (binding) (load "X") (add)

If a sequence contains no instruction,
the sequence is represented as follows:

::

  Ōłģ

.. _call_stack:

Call stack
----------

An :dfn:`abstract stack machine` operates on its :dfn:`call stack`.

A call stack consists of a sequence of instructions,
and the sequence of instructions is updated every time a reduction rule is applied.

A call stack is represented like the following:

::

  ^ (str "Foo") (dup) (remove) $

Here, ``^`` is the mark of the top of the call stack,
and ``$`` is the mark of the bottom of the call stack.

Reduction rules
---------------

A reduction rule is written like the following:

::

  (val V) (dup) ŌåÆ (val V) (val V)

It means that if there is a sub-sequence of instructions ``(val V) (dup)`` in the call stack,
the reduction rule is applied, and the sub-sequence is updated to ``(val V) (val V)``.
Note that ``V`` is a metavariable.
In this case, the metavariable ``V`` represents a Kink val,
because ``val`` instruction takes a Kink val as the operand.

Additional conditions of a reduction rule can be given using comment lines which start with ``#``.

If there are multiple reduction rules which can be applied for a call stack,
the reduction rule which can be applied to the leftmost sequence is chosen.

Metavariable
------------

In this chapter, metavariables of the following formats are used to represent 
an abstract instruction, a sequence of abstruct instructions, and other data.

``_snake_case_with_underscore_as_prefix_and_suffix_``
  A sequence of abstract instructions.

``Capitalized_snake_case``
  An abstract instruction, a val, a str, a num, or a location, depending on the context.

Initial state
=============

Evaluation of an abstract stack machine starts invoking a fun with no args.
When evaluation is started with a fun val ``Fun``,
the initial state of the call stack is:

::

  ^ (val Fun) (nada) (emptyvec) (call "..root..") $

Terminal state
==============

If no reduction rule can be applied to the call stack,
the call stack is said to be in a :dfn:`terminal state`.

The terminal state should be represented in one of the following states.

::

  ^ (val V) $

This terminal state means that the abstract stack machine terminates returining a val.

::

  ^ (uncaught Exc) $

This terminal state means the abstract stack machine terminates with an uncaught exception.

Abstract instructions
=====================

val instruction
---------------

``val`` instruction places a Kink val on the call stack.

.. rubric:: syntax

::

  (val V)

There is no reduction rule for ``val`` instruction.

A single ``val`` instruction can be a terminal state of the :dfn:`abstract stack machine`.

uncaught instruction
--------------------

``uncaught`` instruction means an exception which is not caught within the :dfn:`abstract stack machine`.

.. rubric:: syntax

::

  (uncaught Exc)

``Exc`` is an exception val.

.. rubric:: reduction rule

::

  (uncaught Exc) Any ŌåÆ (uncaught Exc)

* ``Any`` is any instruction

This rule means that the uncaught exception is propagated until the terminal state.

num instruction
---------------

``num`` instruction produces a num val.

.. rubric:: syntax

::

  (num Num)

``Num`` parameter is a num.

.. rubric:: reduction rule

::

  (num Num)

  ŌåÆ (val Num_val)
  # where `Num_val` is a num val converted from `Num`.

str instruction
---------------

``str`` instruction produces a str val.

.. rubric:: syntax

::

  (str Str)

``Str`` parameter is a str.

.. rubric:: reduction rule

::

  (str Str)

  ŌåÆ (val Str_val)
  # where `Str_val` is a str val converted from `Str`

nada instruction
----------------

``nada`` instruction produces ``nada`` val.

.. rubric:: syntax

::

  (nada)

.. rubric:: reduction rule

::

  (nada)

  ŌåÆ (val Nada_val)
  # where `Nada_val` is `nada` val

fun instruction
---------------

``fun`` instruction produces a :term:`function` val.

.. rubric:: syntax

::

  (fun _insns_)

``_insns_`` is a sequence of instructions.

.. rubric:: reduction rule

::

  (val Enclosing_binding) (fun _insns_)
  # where
  # ŌĆó `Enclosing_binding` is a binding val.

  ŌåÆ (val Fun)
  # where
  # ŌĆó `Fun` is a fun val with instructions copied from `_insns_`,
  #   replacing every occurrence of (enclosingbinding) by (val Enclosing_binding).

emptyvec instruction
--------------------

``emptyvec`` instruction produces a newly created vec val with no element.

.. rubric:: syntax

::

  (emptyvec)

.. rubric:: reduction rule

::

  (emptyvec)

  ŌåÆ (val Empty_vec)
  # where `Empty_vec` is a newly created vec val with no element

add instruction
---------------

``add`` instruction produces a newly created vec val adding an element to
the back of an existing vec.

.. rubric:: syntax

::

  (add)

.. rubric:: reduction rule

::

  (val Vec) (val Elem) (add)
  # where `Vec` is a vec val

  ŌåÆ (val New_vec)
  # where `New_vec` is a newly created vec val containing the elements of `Vec`
  # at the front, and `Elem` at the back.


concat instruction
------------------

``concat`` instruction produces ``doconcat`` and ``trace`` instructions.

.. rubric:: syntax

::

  (concat)

.. rubric:: reduction rules

::

  (concat)

  ŌåÆ (doconcat) (trace "" Location)
  # where
  # ŌĆó `Location` is the location of the implicit location of the `concat` instruction.

Note that the trace is used when an exception occurs when
a reduction rule to ``doconcat`` is applied.

doconcat instruction
--------------------

``doconcat`` instruction produces a newly created vec val concatenating
two vec vals.

.. rubric:: syntax

::

  (doconcat)

.. rubric:: reduction rules

::

  (val X) (val Y) (doconcat)
  # where
  # ŌĆó `X` is a vec val
  # ŌĆó `Y` is a vec val

  ŌåÆ (val New_vec)
  # where `New_vec` is a newly created vec val
  # concatenating `X` at the front, and `Y` at the back.

::

  (val X) (val Non_vec) (doconcat)
  # where
  # ŌĆó `X` is a vec val
  # ŌĆó `Non_vec` is not a vec val

  ŌåÆ (str Message) (traces) (exception) (raise)
  # where
  # ŌĆó `Message` is an error message.

dup instruction
---------------

``dup`` instruction duplicates the preceding val on the call stack.

.. rubric:: syntax

::

.. rubric:: reduction rule

::

  (val V) (dup) ŌåÆ (val V) (val V)

flip instruction
----------------

``flip`` instruction swaps the preceding two vals on the call stack.

.. rubric:: syntax

::

  (flip)

.. rubric:: reduction rule

::

  (val A) (val B) (flip) ŌåÆ (val B) (val A)

remove instruction
------------------

``remove`` instruction removes the preceding val from the call stack.

.. rubric:: syntax

::

  (remove)

.. rubric:: reduction rule

::

  (val V) (remove) ŌåÆ Ōłģ

varref instruction
------------------

``varref`` instruction produces a :doc:`varref val </api/kink-VARREF>`.

.. rubric:: syntax

::

  (varref Symbol)

``Symbol`` is a str.

.. rubric:: reduction rule

::

  (val Owner) (varref Symbol)

  ŌåÆ (val Varref)
  # where `Varref` is a varref val whose owner is `Owner`, and whose symbol is `Symbol`.

load instruction
-----------------

``load`` instruction does :term:`variable-load operation` for a variable.

.. rubric:: syntax

::

  (load Symbol)

``Symbol`` is a a str.

.. rubric:: reduction rules

::

  (val Owner) (load Symbol)
  # where the content of variable (`Owner`, `Symbol`) is not empty

  ŌåÆ (val Content)
  # where `Content` is the content of the variable (`Owner`, `Symbol`)


::

  (val Owner) (load Symbol)
  # where the content of variable (`Owner`, `Symbol`) does not have a value.

  ŌåÆ (str Message) (traces) (exception) (raise) (trace "" Location)
  # where
  # ŌĆó `Message` is an error message.
  # ŌĆó `Location` is the implicit location of the `load` instruction.

checkfun instruction
--------------------

``checkfun`` instruction ensures that the preceding val is a fun val.

.. rubric:: syntax

::

  (checkfun)

.. rubric:: reduction rules

::

  (val Fun) (checkfun)
  # where
  # ŌĆó `Fun` is a fun val.

  ŌåÆ Ōłģ

::

  (val Not_fun) (checkfun)
  # where
  # ŌĆó `Not_fun` is not a fun val.

  ŌåÆ (str Message) (traces) (exception) (raise) (trace "" Location)
  # where
  # ŌĆó `Message` is an error message.
  # ŌĆó `Location` is the implicit location of the `checkfun` instruction.

call instruction
----------------

``call`` instruction produces a sequence of instructions
which calls a fun.

.. rubric:: syntax

::

  (call Symbol)

``Symbol`` is a str.

.. rubric:: reduction rules

::

  (val Fun) (val Recv) (val Args) (call Symbol) (tail)
  # where
  # ŌĆó `Fun` is a fun val.
  # ŌĆó `Args` is a vec val.

  ŌåÆ (val Fun) (val Recv) (val Args) (tailcall) (tailtrace Symbol Location)
  # where
  # ŌĆó `Location` is the implicit location of the `call` instruction.

::

  (val Fun) (val Recv) (val Args) (call Symbol) Non_tail
  # where
  # ŌĆó `Fun` is a fun val.
  # ŌĆó `Args` is a vec val.
  # ŌĆó `Non_tail` is an instruction which is not `tail`

  ŌåÆ (val Fun) (val Recv) (val Args) (docall) (trace Symbol Location) Non_tail
  # where
  # ŌĆó `Location` is the implicit location of the `call` instruction.

.. _tailcall_instruction:

tailcall instruction
--------------------

``tailcall`` instruction sets up tail traces and invocation of a fun.
A function call done by ``tailcall`` is said to be a :dfn:`tail call`.

.. rubric:: syntax

::

  (tailcall)

.. rubric:: reduction rule

::

  (tailcall) _tail_traces_
  # where
  # ŌĆó `_tail_traces_` is a sequence of 0 or more `tailtrace` instructions.

  ŌåÆ (docall) _snipped_tail_traces_
  # where
  # ŌĆó `_snipped_tail_traces_` is a sequence of `tailtrace` instructions,
  #   snipped from `_tail_traces_` following the the rule below.

.. rubric:: tail trace snipping rule

This specification requires that consecutive tail calls do not result in stack overflow.
Such requirement is called :term:`tail call elimination`.
To guarantee it, the sequence of ``tailtrace`` instructions is snipped as follows:

* Each ``tailtrace`` instruction can be remvoed.
* Each ``tailtrace`` instruction can be converted to
  ``(tailtrace "" (program name="", program text="", pos index in the program text=0))``
* The length of the sequence must be kept less than or equal to
  the constant maximum number which the runtime defines.

docall instruction
------------------

``docall`` instruction invokes a fun.

.. rubric:: syntax

::

  (docall)

.. rubric:: reduction rule

::

   (val Fun) (val Recv) (val Args) (docall)
   # where
   # ŌĆó `Fun` is a fun val.
   # ŌĆó `Args` is a vec val.

   ŌåÆ (val Recv) (val Args_copy) _fun_insns_ (tail)
   # where
   # ŌĆó `_fun_insns_` is the insns within `Fun`.
   # ŌĆó `Args_copy` is a newly created vec val copying the elements of `Args`.

traces instruction
------------------

``traces`` instruction makes a vec of trace vals
from ``trace`` and ``tailtrace`` instructions in the call stack.

.. rubric:: syntax

::

  (traces)

.. rubric:: reduction rules

::

  (traces) _insns_ $
  # where `_insns_` is a sequence of instructions.

  ŌåÆ (val Traces) _insns_ $
  # where
  # ŌĆó `Traces` is a newly created vec val of trace vals,
  #   converted from `trace`, `tailtrace`, and `delimit` instructions in `_insns_` as follows,
  #   reverting the order.
  #   ŌĆó From a `trace` instruction, a non-tail trace val is produced.
  #   ŌĆó From a `tailtrace` instruction, a tail trace val is produced.
  #   ŌĆó From a `delimit` instruction, a non-tail trace val is produced with symbol="..kont tag..",
  #     and location=("", "", 0).


exception instruction
---------------------

``exception`` instruction produces an :term:`exception` value.

.. rubric:: syntax

::

  (exception)

.. rubric:: reduction rule

::

  (val Message) (val Traces) (exception)
  # where
  # ŌĆó `Message` is a str val
  # ŌĆó `Traces` is a vec of trace vals

  ŌåÆ (val Exc)
  # where `Exc` is an exception val made from `Message` and `Traces`

raise instruction
-----------------

``raise`` instruction raises an exception.

.. rubric:: syntax

::

  (raise)

.. rubric:: reduction rules

::

  (val Exc) (raise) _no_escape_ (val Escape_tag) (delimit)
  # where
  # ŌĆó `Escape_tag` is the escape tag (see kink/KONT_TAG.escape_tag).
  # ŌĆó `_no_escape_` is a sequence of instructions
  #   which does not contain (delimit) directly preceded by (val Escape_tag).

  ŌåÆ (val Fun) (val Escape_tag) (delimit)
  # where `Fun` is a fun val like {(:cont_raised :cont_jump) cont_raised(Exc) }

::

  (val Exc) (raise) _no_escape_ $
  # where
  # ŌĆó `_no_escape_` is a sequence of instructions
  #   which does not contain (delimit) directly preceded by (val Escape_tag).

  ŌåÆ (uncaught Exc) _no_escape_ $

setbinding instruction

``setbinding`` instruction sets the current binding.

.. rubric:: syntax

::

  (setbinding)

.. rubric:: reduction rule

  (val Binding) (setbinding) _insns_ (tail)
  # where
  # ŌĆó `Binding` is a binding val.
  # ŌĆó `_insns_` is a sequence of instructions
  #   which does not contain `tail` instruction.

  ŌåÆ _converted_insns_ (tail)

  #   copied from `_insns_`,

storerecvargs instruction
-------------------------

``storerecvargs`` instruction sets ``_Recv`` and ``_Args`` vars of the current binding.

.. rubric:: syntax

::

  (storerecvargs)

.. rubric:: reduction rule

::

  (val Recv) (val Args) (val Binding) (storerecvargs)
  # where
  # ŌĆó `Args` is a vec val.
  # ŌĆó `Binding` is a binding val.

  ŌåÆ Ōłģ

When this reduction rule is applied,
``Recv`` is assigined to ``_Recv`` variable of ``Binding``,
and ``Args`` is assigined to ``_Args`` variable of ``Binding``.

  (tail)

.. _hostcall_instruction:

hostcall instruction
--------------------

``hostcall`` instruction invokes a :term:`host procedure`.

A :dfn:`host procedure` is an abstract unit of information
to specify a procedure within the :term:`host system`,
such as a function address, closure, object reference.
A host procedure should take a single value as the argument,
and return a single value as the result.

.. rubric:: syntax

::

  (hostcall Host_proc)

``Host_proc`` is a :dfn:`host procedure`.

.. rubric:: reduction rule

::

  (val Arg) (hostcall Host_proc)

  ŌåÆ (val Result)
  # where
  # ŌĆó `Result` is the result of `Host_proc` with the argument `Arg`.

canshift instruction
--------------------

``canshift`` instruction tests whether
the call stack is delimited by a :term:`delimited continuation tag`.

.. rubric:: syntax

::

  (canshift)

.. rubric:: reduction rules

::

  (val Kont_tag) (canshift) _insns_ (val Kont_tag) (delimit)
  # where
  # ŌĆó `Kont_tag` is a delimited continuation tag.
  ŌåÆ (val True) _insns_ (val Kont_tag) (delimit)
  # where
  # ŌĆó `True` is true val.

  (val Kont_tag) (canshift) _no_kont_tag_ $
  # where
  # ŌĆó `Kont_tag` is a delimited continuation tag.
  # ŌĆó `_no_kont_tag_` is a sequence of instructions
  #   which does not contain (delimit) directly preceded by (val Kont_tag).
  ŌåÆ (val False) _no_kont_tag_ $
  # where
  # ŌĆó `False` is false val.

.. _kont_instruction:

kont instruction
----------------

``kont`` instruction makes a :term:`delimited continuation`
function from the call stack.

A :dfn:`delimited continuation`
is a :term:`function` to resume the evaluation between ``kont`` instruction
and ``delimit`` instruction, or in other words,
from an invocation of :ref:`Kont_tag.shift <api-35db7c321b93a22b6e6164fab36bf940>`
to the end of an invocation of :ref:`Kont_tag.reset <api-9877213663340a1e14488dc493151c3a>`.
Construction of a delimited continuation is specified below.

.. rubric:: syntax

::

  (kont)

.. rubric:: reduction rule

::

  (val Kont_tag) (kont) (val Kont_tag) (abort) _insns_ (val Kont_tag) (delimit)
  # where
  # ŌĆó `Kont_tag` is a delimited continuation tag.
  # ŌĆó `_insns_` is a sequence of instructions which does not contain (delimit)
  #   directly preceded by (val Kont_tag).

  ŌåÆ (val Kont_fun) (val Kont_tag) (abort) _insns_ (val Kont_tag) (delimit)
  # where
  # ŌĆó `Kont_fun` is a delimited continuation created as follows:

.. rubric:: making of a delimited continuation

In the previous reduction rule, ``Kont_fun`` is a function
which takes a single argument.
When the function is called,
at the tail of the invocation,
the following sequence of instructions is evaluated:

::

  {push the argument of Kont_fun}
  _no_delimit_
  (val Kont_tag)
  (delimit)

abort instruction
-----------------

``abort`` instruction moves to the corresponding ``delimit``.

.. rubric:: syntax

::

  (abort)

.. rubric:: reduction rule

::

  (val Abort_fun) (val Kont_fun) (val Kont_tag) (abort) _insns_ (val Kont_tag) (delimit)
  # where
  # ŌĆó `Abort_fun` is a fun val.
  # ŌĆó `Kont_fun` is a fun val.
  # ŌĆó `Kont_tag` is a delimited continuation tag.
  # ŌĆó `_insns_` is a  sequence of instructions which does not contain (delimit)
  #   directly preceded by (val Kont_tag).

  ŌåÆ (val Abort_fun) (nada) (emptyvec) (val Kont_fun) (add) (call "call") (val Kont_tag) (delimit)

.. rubric:: syntax

.. rubric:: reduction rule

::

  (val X) (val Kont_tag) (delimit) ŌåÆ (val X)

Resource error
==============

The following reduction rule can be applied anytime,
when any type of resource error happens,
for example,
when the length of the sequence of instructions of the call stack becomes too long (stack overflow).

.. rubric:: reduction rule

::

  ^

  ŌåÆ ^ (str Message) (traces) (exception) (raise)
  # where
  # ŌĆó `Message` is an error message.
